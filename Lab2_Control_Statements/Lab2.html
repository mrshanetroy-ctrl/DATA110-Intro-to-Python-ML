Q1: File Differences¶

q1.Why do the file types (.txt, .csv, .xlsx) differ when loading into Python?

The three files do not contain the same structure or formatting, which causes differences when loading them:

The TXT file loads with 715 rows and 31 columns, showing many extra empty or unnamed columns.

The CSV file loads with 715 rows and only 3 columns, meaning the file uses a different delimiter (likely tab), so it does not split the data correctly.

The Excel file loads cleanly with 715 rows and 18 properly labeled columns.

This shows that file type affects:

How the data is separated (delimiters)

Whether column names are preserved

How many variables are correctly recognized

Conclusion: Yes, the source data format makes a difference. The Excel file is the most complete and properly formatted, which is why it was used for the remaining analysis.

Q3: Describe() Interpretation¶

q3.What does the describe() output show, and how do you interpret it?

The describe() function provides statistical summaries for each numerical column:

count – number of valid values

mean – average

std – standard deviation (variability)

min/max – range

25%, 50%, 75% – quartiles

Key insights:

Age averages around 24, ranging from 13 to 81.

Photos, Tags, Likes, Friends show very high maximum values and large standard deviations, indicating some users are extremely active.

Many columns such as Gender, Emp, Profile, Orientation are binary (0/1).

Conclusion: describe() reveals the distribution characteristics and highlights outliers and skewness in activity-related variables.
